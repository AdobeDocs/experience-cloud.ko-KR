# 데스크탑 애플리케이션 {#desktop-applications}

데스크탑 애플리케이션은 직접 REST API를 호출하여 경험 롤아웃과 통합됩니다. 데스크톱 클라이언트용 **기능 API V2**&#x200B;을(를) 사용합니다.

전체 API 참조는 이 안내서의 기능 API 섹션에서 **GET 기능 API V2**&#x200B;를 참조하십시오.

## 통합 요구 사항 {#requirements}

데스크톱 클라이언트는 다음 작업을 수행해야 합니다.

* API 응답에서 반환된 **TTL 값을 적용합니다**. TTL은 클라이언트가 API를 다시 호출하기 전에 기능 플래그 데이터를 캐시해야 하는 시간을 정의합니다. 기능 플래그가 제공되지 않을 때 이전 애플리케이션 버전에서 최대 절전 모드를 올바르게 입력하도록 이 동작의 유효성을 검사하는 테스트 사례를 구현합니다.
* **HTTP 오류 시나리오를 정상적으로 처리**. API를 일시적으로 사용할 수 없는 경우 애플리케이션이 계속 작동하도록 대체 기능 집합에 빌드합니다.
* 위의 TTL 및 오류 처리 요구 사항을 모두 다루는 **자세한 통합 테스트 계획을 유지 관리합니다**.

## 애플리케이션 ID {#app-id}

데스크톱 클라이언트는 표준 클라이언트 ID 대신 API를 호출할 때 **제품 코드 및 제품 버전**&#x200B;을 응용 프로그램 식별자로 사용할 수 있습니다.
