# 기능 플래그 가져오기 {#import-feature-flags}

경험 롤아웃을 사용하면 기능 플래그를 하위 환경(예: 스테이지)에서 상위 환경(예: 프로덕션)으로 가져올 수 있습니다. 이렇게 하면 플래그 구성을 수동으로 다시 생성하지 않아도 되고 환경 간 구성 드리프트가 발생할 위험이 줄어듭니다.

## 전제 조건 {#prerequisites}

가져오기 워크플로우를 사용하려면 애플리케이션 인스턴스가 여러 환경에 연결되어 있어야 합니다. [환경을 응용 프로그램에 연결](associate-environments.md)을 참조하십시오.

## 1단계: 대상 환경 및 애플리케이션으로 이동 {#step-1}

**대상** 환경, 즉 플래그를 *대상*&#x200B;에 가져올 환경에 콘솔에 로그인합니다. 기능 플래그 페이지의 응용 프로그램 드롭다운에서 플래그를 가져올 응용 프로그램을 선택합니다.

>[!IMPORTANT]
>
>현재 환경과 선택한 응용 프로그램은 소스가 아닌 **대상**&#x200B;이어야 합니다. 예를 들어, 스테이지에서 프로덕션으로 플래그를 가져오려면 프로덕션 콘솔에 로그인하고 프로덕션 애플리케이션을 선택합니다.

## 2단계: 가져오기 대화 상자 열기 {#step-2}

**기능 플래그 가져오기**&#x200B;를 선택하십시오. 소스 환경 및 애플리케이션을 표시하는 대화 상자가 열리고 애플리케이션에 대해 구성된 연결된 환경을 기반으로 미리 채워집니다. 필요한 경우 대화 상자의 드롭다운에서 소스 환경과 애플리케이션을 변경할 수 있습니다.

## 3단계: 가져올 기능 플래그 선택 {#step-3}

소스 환경의 기능 플래그 목록에서 가져올 플래그를 선택합니다. 한 번에 하나, 여러 개 또는 모든 플래그를 선택할 수 있습니다.

## 4단계: 기존 플래그 처리(필요한 경우) {#step-4}

대상 환경에 동일한 키를 가진 기능 플래그가 이미 있는 경우 경험 롤아웃에서 덮어쓰기 여부를 확인하라는 메시지가 표시됩니다. 덮어쓰기가 대상 플래그의 설정을 소스의 설정으로 대체하므로 확인하기 전에 기존 플래그 구성을 검토하십시오.

## 중요 정보 {#important-notes}

기능 플래그를 가져올 때는 다음 사항에 유의하십시오.

* 가져온 플래그는 원본 환경의 상태와 관계없이 항상 대상 환경의 **OFF** 상태로 설정됩니다. 가져온 후에는 수동으로 활성화해야 합니다.
* 플래그가 원본 환경의 미래 날짜 및 시간에 활성화되도록 예약된 경우 해당 일정은 **이월되지 않음**&#x200B;입니다. 필요한 경우 대상 환경에서 새 일정을 설정해야 합니다.
