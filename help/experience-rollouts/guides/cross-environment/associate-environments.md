# 환경에 애플리케이션 연결 {#associate-environments}

환경 간에 애플리케이션 인스턴스를 연결하면 환경 간 가시성과 가져오기 워크플로우를 사용할 수 있습니다. 이를 구성하려면 **관리자** 역할이 있어야 합니다. 역할을 확인해야 하는 경우 [사용자 역할](../teams/user-roles.md)을 참조하세요.

이 방법이 유용한 이유에 대한 배경은 [환경 간 개념](cross-environment-concept.md)을 참조하세요.

## 1단계: 응용 프로그램 설정 열기 {#step-1}

1. 경험 롤아웃 콘솔에 로그인합니다.
2. **관리자 > 응용 프로그램**(으)로 이동합니다.
3. **새 응용 프로그램**&#x200B;을 선택하여 새 앱을 온보딩하거나 기존 응용 프로그램을 선택하여 편집하십시오.

## 2단계: 관련 환경 섹션 구성 {#step-2}

응용 프로그램 양식에서 **관련 환경** 섹션으로 스크롤합니다. 이 섹션의 각 행은 애플리케이션의 배포 환경 중 하나를 정의합니다. 각 환경에 대해 다음 필드를 구성합니다.

| 필드 | 설명 |
|---|---|
| **환경** | 목록에서 표준 환경 식별자인 QA1, QA2, STG1, STG2, PROD1 또는 PROD2를 선택합니다. 이는 환경이 프로덕션인지 아니면 비프로덕션인지를 경험 롤아웃에 알려주고, 콘솔에 사용되는 색상 코딩을 결정합니다. |
| **환경 레이블** | 팀에서 참조하는 이 환경의 사용자 지정 이름(예: `Dev`, `Staging` 또는 `Production`)입니다. 이는 자유 형식이며 표준 식별자와 일치시킬 필요가 없습니다. |
| **경험 롤아웃 환경** | 이 애플리케이션 인스턴스가 호스팅되는 플랫폼 환경(스테이지 또는 프로덕션)입니다. 현재 있는 콘솔을 기반으로 미리 채워집니다. |
| **응용 프로그램 ID** | 이 환경에 있는 애플리케이션 인스턴스의 클라이언트 ID입니다. 드롭다운에서 응용 프로그램을 선택하면 자동으로 채워집니다. |

## 3단계: 여러 환경에서 애플리케이션 인스턴스 연결 {#step-3}

여러 환경에서 인스턴스를 연결하려면 각 추가 환경에 대해 다음 단계를 수행하십시오.

1. 대상 환경에서 애플리케이션 인스턴스 만들기(예: 스테이지 콘솔에서 QA 앱 만들기).
2. 구성 중인 응용 프로그램으로 돌아가서 **관련 환경**&#x200B;에 새 행을 추가합니다.
3. **응용 프로그램 ID** 드롭다운에서 방금 만든 응용 프로그램 인스턴스를 선택합니다. 환경 태그와 레이블이 자동으로 채워집니다.
4. 저장하려면 **업데이트**&#x200B;를 선택하십시오.

연결할 각 환경에 대해 이 프로세스를 반복합니다(예: 동일한 애플리케이션의 QA, 스테이지 및 프로덕션 인스턴스 연결).

## 중요 정보 {#important-notes}

* QA 및 스테이지 환경은 경험 롤아웃 스테이지 또는 프로덕션 플랫폼 환경에서 호스팅될 수 있습니다.
* 프로덕션 환경(PROD1, PROD2)은 Experience Rollouts 프로덕션 환경에서만 호스팅할 수 있습니다.
* 높은 환경에서만 낮은 환경에 연결할 수 있습니다. 예를 들어 프로덕션에서 스테이지 또는 QA 인스턴스에 연결할 수 있지만 반대는 읽기 전용입니다.
