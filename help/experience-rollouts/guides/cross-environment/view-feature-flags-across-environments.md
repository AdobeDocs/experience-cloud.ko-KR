# 환경 간 기능 플래그 보기 {#view-flags-across-environments}

환경 간에 애플리케이션 인스턴스를 연결하면 Experience Rollouts에서 기능 플래그 목록 페이지에 연결된 모든 환경의 각 기능 플래그 상태를 바로 표시합니다. 이렇게 하면 환경을 전환하지 않고 플래그 상태(예: 스테이지에서 플래그가 켜지고 프로덕션에서 꺼짐)를 비교할 수 있는 단일 보기가 제공됩니다.

## 전제 조건 {#prerequisites}

환경 간 상태가 표시되기 전에 애플리케이션 인스턴스가 여러 환경에 연결되어 있어야 합니다. 설치 지침은 [환경을 응용 프로그램에 연결](associate-environments.md)을 참조하십시오.

## 작동 방식 {#how-it-works}

**기능 및 릴리스 > 기능 플래그**(으)로 이동하여 연결된 인스턴스가 있는 응용 프로그램을 선택하면 목록 페이지에 연결된 각 환경에 대한 추가 열이 표시됩니다. 각 열에는 해당 환경에서 기능 플래그의 현재 상태가 표시됩니다.

기능 플래그는 환경 간에 이름이 아닌 **키**&#x200B;로 일치합니다. 이것은 다음을 의미합니다.

* 키가 동일한 경우 플래그는 각 환경에서 다른 표시 이름을 가질 수 있습니다.
* 각 환경 열에 표시되는 이름은 해당 환경에서 사용되는 이름입니다.

## 사용 사례 {#use-case}

일반적인 워크플로는 구성을 프로덕션으로 승격하기 전에 하위 환경(예: 스테이지)에서 기능 플래그를 개발 및 테스트한 다음 상태를 확인하는 것입니다. 환경 간 가시성을 사용하면 프로덕션 콘솔 내에서 프로덕션으로 가져오기 전에 스테이지에서 플래그가 올바르게 구성되었는지 확인할 수 있습니다.
