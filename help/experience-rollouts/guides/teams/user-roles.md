# 사용자 역할 {#user-roles}

경험 롤아웃은 5개의 역할을 제공합니다. 각 역할은 특정 책임 세트를 위해 설계되었습니다. 역할은 기본적으로 함께 사용할 수 없습니다. 관리자 역할을 가진 구성원은 제품 릴리스 소유자 권한을 자동으로 가지지 않습니다. 광범위한 액세스가 필요한 경우 구성원에게 여러 역할을 할당합니다.

## 사용 가능한 역할 {#roles}

| 역할 | 책임 |
|---|---|
| **관리** | 응용 프로그램을 콘솔에 온보딩하고, 팀 구성원을 관리하며, 액세스 요청을 승인 또는 거부합니다. 팀이 기능 플래그를 사용하기 전에 필수입니다. |
| **제품 릴리스 소유자** | 응용 프로그램의 기능 플래그 및 기능 그룹을 만들고 업데이트합니다. 기능 플래그에 대상을 연결하여 외부 사용자가 기능을 사용할 수 있습니다. |
| **Developer** | 샌드박스 컨텍스트에서 작동하여 다른 사용자에게 영향을 주지 않고 기능 플래그 뒤에 있는 기능을 테스트합니다. 스테이지 및 프로덕션의 개인 대상(예: 특정 사용자 ID 또는 이메일 주소)에 기능 플래그를 노출할 수 있습니다. |
| **기능 관리자** | 팀 간 기능 그룹을 만들고 관리합니다. 다른 팀이 소유한 응용 프로그램 간에 플래그를 조정할 때 이 역할을 사용합니다. |
| **릴리스 관리자** | 다중 팀, 다중 애플리케이션 릴리스를 관리하고 릴리스를 받는 사용자를 정의하는 대상 규칙을 업데이트합니다. |

## 적합한 역할 선택 {#choosing}

다음 지침을 사용하여 역할을 할당합니다.

* **응용 프로그램에 대한 기능 플래그 추가 또는 업데이트** → **제품 릴리스 소유자**
* **다른 사용자에게 영향을 주지 않고 개별적으로 기능 테스트** → **개발자**
* **팀 간 기능 그룹 관리** → **기능 관리자**
* **여러 팀 및 응용 프로그램에서 릴리스 관리** → **릴리스 관리자**
* **응용 프로그램 온보딩 및 팀 구성원 관리** → **관리자**

>[!NOTE]
>
>구성원이 두 개 이상의 역할을 보유할 수 있습니다. 예를 들어 팀의 응용 프로그램을 테스트하고 관리하는 엔지니어는 **개발자** 및 **관리자** 역할을 모두 가져야 합니다.

## 권한 오류 {#permissions-error}

구성원에게 기능 또는 그룹을 업데이트하려고 할 때 &quot;권한이 충분하지 않음&quot; 오류가 발생하면 해당 구성원은 **제품 릴리스 소유자** 역할이 필요합니다. 이 역할을 추가하려면 팀 관리자에게 문의하십시오.
