# 문제 해결 {#troubleshooting}

경험 롤아웃에는 **기능 워크벤치**&#x200B;라는 기본 진단 도구가 포함되어 있어 특정 사용자가 받는 기능 플래그와 그 이유를 정확하게 파악할 수 있습니다. 코드에서 문제를 재현할 필요 없이 예기치 않은 기능 동작에 대한 보고서를 조사할 때 사용합니다.

## 기능 워크벤치 액세스 {#access-workbench}

기능 워크벤치를 열려면 다음 단계를 수행하십시오.

1. 경험 롤아웃 콘솔에 로그인합니다.
2. 상단 메뉴에서 **Workbench > 최종 사용자**(으)로 이동합니다.
3. **기능** 탭을 선택합니다.

## 사용자 ID 입력 {#input-identity}

특정 사용자에 대한 기능 플래그 평가를 조회하려면 워크벤치 양식에 다음 정보를 입력합니다.

* **ID 유형** — 사용 중인 식별자 유형을 선택합니다. 지원되는 유형에는 이메일, GUID 및 방문자 ID가 포함됩니다. **릴리스 플래그**&#x200B;를 직접 입력하여 플래그 값에 따라 사용자에 대해 활성화된 릴리스를 조회할 수도 있습니다.
* **ID 값** — 사용자 식별자를 입력합니다. ID 유형으로 이메일을 선택하는 경우 여러 GUID를 동일한 이메일 주소와 연결할 수 있습니다. 연결된 GUID 간에 전환할 수 있도록 GUID 드롭다운이 제공됩니다.
* **소스 유형 인증** — 통합에 해당되는 경우 선택합니다.
* **컨텍스트 변수** — 대상 규칙 평가에 영향을 주는 컨텍스트 키-값 쌍(예: 국가, 장치 유형)을 선택적으로 제공합니다.
* **팀 및 응용 프로그램** — 기능 플래그를 가져올 팀 및 응용 프로그램을 선택합니다. 현재 팀과 해당 애플리케이션 중 하나는 기본적으로 미리 선택됩니다.

양식을 작성한 후 **기능 가져오기**&#x200B;를 선택합니다.

## 결과 해석 {#interpret-results}

결과는 입력한 ID 유형에 따라 다릅니다.

### 이메일 또는 GUID 조회 {#email-guid-lookup}

이메일이나 GUID로 사용자를 조회하면 결과에 두 개의 섹션이 나타납니다.

**릴리스 플래그**

이 섹션에는 선택한 팀 및 애플리케이션에 대한 세 가지 목록이 표시됩니다.

* **사용** — 이 사용자에 대해 현재 활성 상태인 릴리스입니다.
* **사용 안 함** - 이 사용자에 대해 존재하지만 사용할 수 없는 릴리스입니다.
* **사용되지 않음** — 릴리스가 첨부되지 않은 릴리스 비트입니다.

현재 선택한 팀 및 애플리케이션과 관련된 릴리스가 각 목록의 맨 위에 나타납니다.

**기능 플래그 및 그룹**

이 섹션에는 현재 사용자에 대해 활성화된 선택된 팀과 응용 프로그램의 모든 기능 플래그가 나열됩니다. 각 항목에 다음이 표시됩니다.

* 기능 플래그 키 및 이름
* 기능 그룹(플래그가 1에 속하는 경우)
* 변형(A/B 테스트가 구성된 경우)
* 플래그가 대상으로 연결되어 있는지 여부

**요청/응답 표시**&#x200B;를 선택하여 결과를 생성한 정확한 API 요청 및 응답을 검사할 수도 있습니다.

### 비자가 이메일 또는 비 GUID ID {#non-self-lookup}

입력한 전자 메일이 사용자 전자 메일이 아니거나 GUID가 아닌 ID 형식을 사용하는 경우 **기능 플래그 및 그룹** 섹션만 표시됩니다.

### 릴리스 플래그 조회 {#release-flag-lookup}

ID 유형으로 **릴리스 플래그**&#x200B;를 입력하면 **릴리스 플래그** 섹션만 표시되고 해당 플래그와 연결된 사용자에 대해 활성화 및 비활성화된 릴리스가 나열됩니다.

## 일반적인 문제 {#common-issues}

다음 테이블에서는 워크벤치를 사용하여 일반적인 문제와 문제를 조사하는 방법을 설명합니다.

| 증상 | 확인할 사항 |
|---|---|
| 사용자가 수신해야 하는 기능 플래그를 받지 못했습니다. | 팀 및 응용 프로그램 선택을 확인합니다. 사용자 ID가 대상 규칙 유형(이메일, GUID 등)과 일치하는지 확인합니다. 기능이 **게시** 또는 **전체 롤아웃** 상태인지 확인하십시오. |
| 예기치 않게 모든 사용자에 대해 기능이 활성화되었습니다. | 기능에 적용된 대상 규칙이 없거나 비율이 100%로 설정되어 있는지 확인합니다. 기능이 **전체 롤아웃** 상태인지 확인하십시오. |
| 대상 규칙이 일치하지 않음 | 컨텍스트 변수와 함께 워크벤치를 사용하여 런타임 시 전달된 값이 대상 규칙에 구성된 값과 일치하는지 확인합니다. |
| 기능 플래그 상태는 올바르지만 사용자에게 이전 동작이 표시됩니다. | 애플리케이션에 대해 구성된 TTL을 확인합니다. SDK은 기능 데이터를 캐시하고 구성된 폴링 간격에만 새로 고칩니다. |
