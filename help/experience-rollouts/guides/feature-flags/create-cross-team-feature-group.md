# 팀 간 기능 그룹 만들기 {#create-cross-team-feature-group}

## 전제 조건 {#prerequisites}

팀 간 기능 그룹을 만들기 전에 다음을 확인하십시오.

* 콘솔에 액세스할 수 있습니다. [콘솔에 로그인](../console/log-in-to-the-console.md)을 참조하세요.
* 팀에 속해 있고 애플리케이션이 온보딩되었습니다.
* **기능 관리자** 역할이 있습니다. [사용자 역할](../teams/user-roles.md) 참조
* 포함할 기능 플래그를 만들었습니다. [첫 번째 기능 플래그 만들기](create-your-first-feature-flag.md)를 참조하십시오.

팀 간 기능 그룹을 사용하면 다양한 대상 타겟팅을 사용하는 여러 팀의 애플리케이션에서 기능 플래그를 번들로 연결할 수 있습니다. 릴리스와 달리 완전히 셀프 서비스이며 만들 수 있는 수에 제한이 없습니다. 비교하려면 [릴리스 및 팀 간 기능 그룹](releases-and-cross-team-feature-groups.md)을 참조하세요.

## 1단계: 팀 간 기능 그룹 만들기 {#create}

콘솔의 릴리스 섹션에서 만들기 프로세스를 시작합니다.

1. 콘솔에서 **기능 및 릴리스 > 릴리스**(으)로 이동합니다.
2. **새 기능 그룹(팀 간)**&#x200B;을 선택합니다.

## 2단계: 기본 세부 정보 {#basic-details}

제목, 키, 설명 및 선택적으로 태그를 제공합니다. 다음 옵션을 구성합니다.

* **롤아웃 비율** — 대상자가 기능을 받는 양을 설정합니다.
* **롤아웃 유형** — 수동 또는 자동화를 선택합니다. 자동 롤아웃 작동 방식에 대한 자세한 내용은 [자동 롤아웃](../automated-rollouts/automated-rollout-concept.md)을 참조하십시오.

>[!NOTE]
>
>A/B 테스트는 팀 간 기능 그룹에 대해 지원되지 않습니다. A/B 테스트를 실행하려면 표준 [기능 그룹](create-a-feature-group.md)을(를) 사용합니다(응용 프로그램은 같은 팀에 있어야 함).

## 3단계: 대상 {#audience}

**대상** 탭에서 **대상 규칙**&#x200B;을 사용하도록 설정하고 다음을 구성하세요.

* **세그먼트** — 사전 정의된 대상 세그먼트를 선택합니다.
* **추가 필터** — 프로필 기반 또는 컨텍스트 기반 기준을 직접 추가합니다.
* **응용 프로그램** — 팀이나 다른 팀에서 하나 이상의 응용 프로그램을 선택합니다.

>[!IMPORTANT]
>
>다른 팀에서 애플리케이션을 추가하면 해당 팀의 기능 관리자가 애플리케이션에 대한 기능 플래그를 이 팀 간 기능 그룹에 추가할 수 있는 권한을 부여합니다. 기능 플래그를 직접 추가할 수는 없습니다.

## 4단계: 기능 {#features}

**기능** 탭 아래에 각 응용 프로그램에 대한 상자가 표시됩니다.

* 자체 팀의 응용 프로그램의 경우 기능 플래그를 직접 추가할 수 있습니다.
* 다른 팀의 응용 프로그램의 경우 해당 상자가 회색으로 표시됩니다. 각 팀의 기능 관리자는 해당 플래그를 추가해야 합니다.

>[!IMPORTANT]
>
>기능 플래그는 한 번에 하나의 메서드를 통해서만 제공할 수 있습니다. 팀 간 기능 그룹에 플래그를 추가하면 플래그에 설정된 대상자 또는 백분율 롤아웃이 직접 제거됩니다. 다른 릴리스 또는 기능 그룹에 이미 있는 플래그는 표시되지 않습니다.

## 5단계: 예약(선택 사항) {#schedule}

팀 간 기능 그룹을 향후 날짜 및 시간에 활성화하도록 예약할 수 있습니다. 자세한 내용은 [일정](schedule.md)을 참조하세요.

## 상태 관리 {#states}

팀 간 기능 그룹을 만드는 팀은 이 그룹을 소유하고 해당 상태를 관리할 수 있습니다. 소유 팀의 기능 관리자 멤버는 상태 드롭다운을 사용하여 상태 간에 전환할 수 있습니다.

| 주/도 | 설명 |
|---|---|
| **저장** | 저장되었으며 라이브가 아님. 모든 변경이 허용됩니다. |
| **게시** | 구성된 대상자에 대해 라이브합니다. 기능 관리자는 대상 기준 및 비율 롤아웃을 계속 업데이트할 수 있습니다. |
| **전체 롤아웃** | 모든 사용자가 사용할 수 있습니다. 이 시점 이후에는 대상을 더 이상 제한할 수 없습니다. |
| **기준선** | 이제 기능이 기본값입니다. 팀 간 기능 그룹이 닫혔습니다. |
| **중단** | 롤아웃이 중지됩니다. 이 그룹에서 기능을 받는 사용자가 없습니다. |

## FAQ {#faqs}

**팀 간 기능 그룹의 수에 제한이 있습니까?**
아니요. 릴리스와 달리 제한은 없습니다. 그러나 더 이상 필요하지 않은 그룹을 보관하거나 비활성화합니다.

**릴리스와 팀 간 기능 그룹의 차이점은 무엇입니까?**
자세한 비교는 [릴리스 및 팀 간 기능 그룹](releases-and-cross-team-feature-groups.md)을 참조하세요.
