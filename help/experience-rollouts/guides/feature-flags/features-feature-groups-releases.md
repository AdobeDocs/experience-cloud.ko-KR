# 기능, 기능 그룹 및 릴리스 {#features-feature-groups-releases}

경험 롤아웃에서는 기능 롤아웃을 관리하기 위한 4개의 아티팩트를 제공합니다. 올바른 타겟팅 선택은 롤아웃 범위, 관련된 팀의 수 및 필요한 대상 타겟팅 기능에 따라 다릅니다.

## 네 가지 인공물 {#artifacts}

**기능 플래그**
가장 작은 단위입니다. 한 팀이 소유한 단일 응용 프로그램에 대한 단일 기능을 제어합니다. 정의된 대상에 대해 활성화하거나 비활성화할 수 있습니다.

**기능 그룹**
동일한 팀에 속한 기능 플래그의 컬렉션입니다. 한 팀의 여러 애플리케이션에서 여러 플래그를 한 단위로 관리할 수 있습니다.

**팀 간 기능 그룹**
여러 팀 및 애플리케이션에서 기능 그룹 기능을 확장합니다. 셀프서비스 이며 다양한 대상 기준을 지원하지만 캐싱은 지원하지 않습니다.

**릴리스**
여러 팀 및 애플리케이션에서 대규모의 조정된 롤아웃을 위해 설계되었습니다. 인증 토큰 기반 대상 타깃팅을 사용합니다. 서버 SDK에서 캐싱을 지원합니다.

## 비교 {#comparison}

| | 기능 플래그 | 기능 그룹 | 크로스 팀 기능 그룹 | 릴리스 |
|---|---|---|---|---|
| 대상을 관리하는 데 필요한 **역할** | 제품 릴리스 소유자 | 제품 릴리스 소유자 | 기능 관리자 | 릴리스 관리자 |
| **응용 프로그램** | 단일 | 다중(동일한 팀) | 다중(팀 간) | 다중(팀 간) |
| **팀** | 단일 | 단일 | 크로스 팀 | 크로스 팀 |
| **풍부한 대상 기준** | ✓ | ✓ | ✓ | 제한적 |
| **롤아웃 비율** | 모든 대상 기준과 결합 | 모든 대상 기준과 결합 | 모든 대상 기준과 결합 | 고정 대상 기준과 결합 |
| **A/B 테스트** | ✓ | ✓ | ✗ | ✗ |
| **서버 SDK에서 캐싱** | 기본 켜기/끄기 플래그만 | 캐싱 없음 | 캐싱 없음 | 모든 릴리스가 로컬에서 캐시됨 |
| **셀프 서비스** | ✓ | ✓ | ✓ | 지원 요청 필요 |
| **감사 추적** | ✓ | ✓ | ✓ | ✓ |

## 각 항목 사용 시기 {#when-to-use}

| 시나리오 | 사용 |
|---|---|
| 하나의 애플리케이션에 대해 단일 기능 테스트 또는 롤아웃 | **기능 플래그** |
| 동일한 팀에서 여러 기능을 조정하면서 동시에 실행 | **기능 그룹** |
| 다양한 타깃팅을 사용하여 다양한 팀의 애플리케이션 간 기능 조정 | **팀 간 기능 그룹** |
| SDK 수준 캐싱을 사용하여 여러 팀에서 대규모 조정 릴리스 | **릴리스** |
