# Analytics {#analytics}

Experience Rollouts는 기능 플래그, 기능 그룹, 팀 간 기능 그룹 및 릴리스에 대한 내장된 분석을 제공합니다. Analytics 대시보드를 사용하여 롤아웃에 참여하는 사용자 수와 변형 및 컨트롤 그룹을 비교하는 방법을 이해할 수 있습니다.

## 분석 활성화 {#enable}

Analytics는 두 가지 수준에서 활성화되어야 합니다.

1. **응용 프로그램 수준** - Experience Rollouts 지원에 문의하여 응용 프로그램에 대한 분석을 사용하도록 설정하십시오.
2. **기능 플래그 수준** — 응용 프로그램에 대해 분석이 활성화되면 추적할 각 기능 플래그의 **기본 세부 정보** 탭에서 **분석 사용** 확인란을 선택하십시오.

>[!NOTE]
>
>Analytics는 기본적으로 애플리케이션당 최대 20개의 기능 플래그에 대해 활성화할 수 있습니다. 이 제한을 늘려야 하는 경우 지원 센터에 문의하십시오.

## 분석 대시보드 보기 {#dashboard}

분석이 활성화되면 애플리케이션의 모든 기능 플래그, 기능 그룹 및 릴리스가 데이터 추적을 시작합니다. 분석할 기능 플래그, 기능 그룹 또는 릴리스에서 **결과**&#x200B;을(를) 선택하여 대시보드에 액세스합니다.

대시보드가 표시됩니다.

* **참가자** — 기능에 적합한 총 사용자 수(변형 + 컨트롤 그룹 결합)
* **컨트롤 그룹** — 컨트롤 그룹에 할당된 사용자 수(기본 환경을 받은 사용자)
* **일 수준 그래프** — 시간에 따른 변형 및 컨트롤 그룹의 등록을 보여 주는 일별 선 차트입니다. 표식은 기능 플래그 구성이 업데이트된 시기를 나타냅니다
* **변형 수준 분석** - 컨트롤 그룹 및 각 변형에 등록된 사용자의 누적 수

기능 그룹 및 릴리스의 경우 **결과** 드롭다운을 선택하여 응용 프로그램을 선택하고 해당 응용 프로그램에 대한 분석을 봅니다. Analytics는 활성화된 애플리케이션에서만 사용할 수 있습니다.
