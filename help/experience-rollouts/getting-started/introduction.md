# 경험 롤아웃 소개 {#introduction}

Adobe Experience Rollouts는 팀이 기능을 보는 사람과 시기를 정확하게 제어하면서 프로덕션에 이르기까지 기능을 배포할 수 있도록 해 주는 제어된 릴리스 시스템입니다. 기능은 프로덕션에 있을 수 있으며 최종 사용자는 보이지 않을 수 있습니다. 그런 다음 타깃팅된 대상에 대해 점진적으로 켜집니다. 재배포 없이 사용할 수 있습니다.

## 개발에서 프로덕션에 이르기까지 {#dev-to-prod}

여정 롤아웃은 초기 개발 단계부터 일반 출시까지 기능의 전체 롤아웃을 지원합니다.

1. **개발자 테스트** — 개발자는 기능 플래그를 만들고, 코드를 환경에 배포하며, 자체 세션만 테스트합니다. 다른 사용자는 영향을 받지 않으며 분기가 필요하지 않습니다.

2. **타깃팅된 미리 보기** - 제품 소유자가 대상을 기능 플래그로 연결하여 정의된 사용자 하위 집합(예: Beta 참가자 또는 특정 지역)에서 유효성 검사 및 피드백을 위해 해당 기능을 사용할 수 있도록 합니다.

3. **점진적 롤아웃** — 이 기능은 점진적으로 더 많은 대상자에게 개방되며(1%, 10%, 50%, 100%), 어떤 단계에서든 기능을 일시 중지, 조정 또는 끌 수 있습니다.

4. **일반 가용성** — 유효성 검사가 완료되면 모든 사용자가 기능을 사용할 수 있습니다.

## 핵심 기능 {#capabilities}

경험 롤아웃은 다음을 제공하는 기능 관리 플랫폼입니다.

* **기능 플래그** — 런타임 시 코드를 다시 배포하지 않고 타깃팅된 대상에 대해 모든 기능을 켜거나 끕니다.

* **대상 타깃팅** — 사용자 프로필 데이터, 백분율 기반 규칙, 전자 메일 주소, 전자 메일 도메인, IP 주소 또는 상황별 특성을 사용하여 기능을 보는 사용자를 제어합니다.

* **기능 그룹** — 응용 프로그램 간에 여러 관련 기능 플래그를 번들로 묶고 단일 단위로 관리하여 동일한 대상자에게 일관된 환경이 표시되도록 합니다.

* **릴리스** — 여러 팀 및 응용 프로그램의 기능 플래그를 단일 릴리스 이벤트로 그룹화하여 대규모 팀 간 롤아웃을 조정합니다.

* **점진적 롤아웃** - 위험을 줄이고, 피드백을 수집하고, 백엔드 로드를 관리하기 위한 단계적 기능 제공.

* **스위치 중단** — 문제가 감지되면 코드를 변경하거나 다시 배포하지 않고 기능을 즉시 해제합니다.
